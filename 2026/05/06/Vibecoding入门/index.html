Vibecoding: 在 AI 时代，将编码化为心流艺术

发表于2026-05-06|更新于2026-05-06|X-Lab notes
|浏览量:

What is Vibecoding?

Vibecoding 是一种基于生成式 AI 的新型编程范式。它强调通过自然语言描述“意图”和“感觉”（Vibe），而非手动编写具体的逻辑语法。在这种模式下，开发者更像是系统的“导演”，负责决策和方向引导，而 AI 则是负责具体实现的“执行制片”。

核心价值 (Why it Matters?)

  • 心流连续性：减少了查阅文档、调试微小语法错误带来的干扰，保持思维的连贯。
  • 十倍生产力：在原型构建和 UI 开发阶段，速度远超传统手写。
  • 降低技术债门槛：优秀的 AI Agent 工具可以自动处理复杂的重构，降低了重构的心理负担。

主流ai工具

  • cursor 基于vscode的IDE工具，具有独特的composer模式
  • vscode+copilot 提供丝滑的代码补全体验，开通教育优惠后比较有性价比
  • claude code 最火爆的命令行编程工具，据说接claude模型体验最好，个人接入Deepseek v4感觉体验一般
  • codex openAI开发的编程工具，类似于cc，不过我还没有使用过

最佳实践建议

  1. 意图原子化：虽然叫 Vibecoding，但描述意图时越清晰、越细化，AI 的产出质量越高。
  2. 小步快跑：每次只让 AI 修改一个功能点，观察效果后再进行下一步。
  3. 重视 Context：学会使用 @Files 或 .cursorrules 等机制为 AI 提供准确的项目上下文。
  4. 人类审计：Vibe 是方向，逻辑是基石。关键的业务逻辑仍需人类进行最后的代码审计。

实践应用

用VS Code+GitHub Copilot和接入 DeepSeek V4 的 Claude Code两个工具联合开发了一个兼具日程表、待办事项、打卡记录和记账功能的 app（bug 还有点多）。网页端可以访问 ToDoListAPP，安卓或鸿蒙系统可以访问 APK 下载目录 下载安装包试用（这个 bug 少一点）。
