计算机网络、互联网与 Web 渲染原理

发表于2026-05-15|更新于2026-05-15|X-Lab notes
|浏览量:

一、计算机网络到底是什么

计算机网络是由多台计算机、服务器、路由设备和传输介质组成的通信系统，其核心任务是让不同设备之间能够交换数据。

网络通信需要解决以下问题：

  • 目标地址怎么找到。
  • 数据怎么一段段传过去。
  • 中途如果丢了怎么办。
  • 两台设备说的话怎么才能互相听懂。

计算机网络中的三个基础概念如下：

  • 分层：将复杂通信过程拆分为多个层次，每一层只处理一类问题。
  • 协议：通信双方必须遵守的规则集合，用于规定数据格式、交互方式和处理流程。
  • 封装：高层数据在传输过程中会被逐层加入协议头和协议尾，形成适合下层传输的数据单元。

计算机网络的学习重点是明确“数据在每一层如何表示、如何传递、如何被处理”。

二、互联网和普通网络有什么关系

计算机网络是广义概念，互联网是其中规模最大、覆盖最广的一种实现形式。

互联网由多个局域网、广域网和自治系统通过路由器互联形成，并以 TCP/IP 协议族作为主要通信基础。学校局域网、公司内网、家庭网络和运营商网络都可以通过接入设备连接到互联网。

网页访问、即时通信、视频传输、在线游戏、云存储同步等行为，本质上都是通过互联网进行数据交换。

三、分层模型怎么理解

分层模型的作用是划分职责边界，使通信系统的设计、实现和排错都更清晰。

1. 先记住 TCP/IP 四层

TCP/IP 四层模型是互联网最常用的分层方式：

  1. 网络接口层：负责局部链路内的数据传输，包括以太网、Wi-Fi、PPP 等技术。
  2. 网际层：负责跨网络寻址与路由转发，核心协议是 IP。
  3. 传输层：负责端到端通信，核心协议包括 TCP 和 UDP。
  4. 应用层：负责面向具体应用的协议，例如 HTTP、HTTPS、DNS、SMTP、SSH。

OSI 七层模型主要用于理论描述和教学理解。实际互联网更常按 TCP/IP 四层进行分析。

2. 一个具体例子

浏览器访问网页时，各层的职责如下：

  • HTTP 负责定义浏览器和服务器之间的请求响应格式。
  • TCP 负责保证数据尽可能可靠地到达对端。
  • IP 负责根据目标地址完成跨网络转发。
  • Wi-Fi 或网线负责在本地链路上传输比特流。

分层的意义在于：上层只依赖下层提供的能力，而不需要关心下层的具体实现方式。

四、IP、MAC、路由分别是什么

1. IP 地址

IP 地址是网络层地址，用于标识主机或网络接口在互联网络中的位置。

IP 地址的作用是让数据包能够在不同网络之间被正确转发。IPv4 采用 32 位地址，常见书写形式为点分十进制；IPv6 采用 128 位地址，主要用于缓解 IPv4 地址枯竭问题。

IP 层关注的是跨网络寻址，不负责局域网内部的最后一次链路投递。

2. MAC 地址

MAC 地址是数据链路层使用的地址，通常用于局域网内识别网卡。

MAC 地址是网卡的链路层标识，用于在同一局域网中定位目标接口。以太网帧在局域网内转发时依赖 MAC 地址完成投递。

3. ARP 和路由

ARP（地址解析协议）的作用是根据已知 IP 地址查询对应的 MAC 地址，前提是目标位于同一局域网或可达的链路范围内。

路由器的作用是在不同网络之间转发数据包。路由器根据路由表选择下一跳，并在转发过程中替换局部链路层地址。

五、DNS 为什么要存在

域名用于提高记忆性和可读性，网络传输真正依赖的是 IP 地址。

DNS（Domain Name System）的作用是将域名解析为 IP 地址，并支持分布式查询和缓存机制。

DNS 解析通常经过以下步骤：

  1. 浏览器查询本地缓存。
  2. 操作系统查询本地缓存。
  3. 如未命中缓存，请求递归 DNS 服务器。
  4. 递归服务器按根域名服务器、顶级域名服务器、权威域名服务器的顺序查询。
  5. 返回域名对应的 IP 地址，并附带缓存时间。

DNS 的核心结果是把“人可读的域名”转换为“机器可通信的 IP 地址”。

六、TCP 和 UDP 怎么选

1. UDP

UDP 是无连接的传输层协议，协议头开销小，传输方式简单，但不保证可靠到达、不保证顺序，也不提供自动重传。

UDP 常用于对实时性要求较高的场景，例如：

  • 语音通话。
  • 直播。
  • 在线游戏。
  • DNS 查询。

2. TCP

TCP 是面向连接的传输层协议，提供可靠传输、按序交付、流量控制和拥塞控制。

TCP 提供的能力包括：

  • 建立连接。
  • 可靠传输。
  • 按顺序收包。
  • 控制发送速度。

TCP 的关键机制主要包括：

  • 三次握手：建立连接，确认双方的发送和接收能力，并同步初始序列号。
  • 四次挥手：关闭连接，TCP 的两个方向需要分别关闭。
  • 序列号和确认号：用于保证数据按序到达并确认接收情况。
  • 超时重传：在数据丢失或未收到确认时重新发送。
  • 滑动窗口：用于控制发送速率，避免接收方被过量数据压垮。
  • 拥塞控制：用于适应网络中的拥塞情况，降低丢包和延迟。

3. 什么时候用谁

是否选择 TCP 或 UDP，主要取决于业务对可靠性、延迟和顺序性的要求。

当业务需要可靠性和完整性时，优先选择 TCP；当业务更重视实时性且可以容忍少量丢包时，通常选择 UDP，并在应用层补充必要的纠错或重传逻辑。

七、HTTP 是什么

HTTP 是应用层协议，浏览器和服务器通过它交换请求与响应。

HTTP 采用请求-响应模型。客户端发起请求，服务器根据请求返回响应。

一次 HTTP 请求通常包括：

  • 请求行：请求方法、路径、版本。
  • 请求头：一些附加信息，比如浏览器信息、缓存信息、Cookie。
  • 请求体：需要提交的数据，比如表单或 JSON。

一次 HTTP 响应通常包括：

  • 状态码：比如成功、重定向、找不到资源。
  • 响应头：告诉浏览器内容类型、缓存策略等。
  • 响应体：真正返回的数据，比如 HTML、图片、JS、JSON。

常见状态码

  • 200：成功。
  • 301：永久重定向，资源地址发生长期变化。
  • 302：临时重定向，资源地址只是暂时变化。
  • 304：资源未修改，浏览器可直接使用缓存。
  • 400：请求语法错误或参数错误。
  • 401：未认证。
  • 403：无权限访问。
  • 404：资源不存在。
  • 500：服务器内部错误。

HTTP 版本也会影响连接方式与性能特征。HTTP/1.1 以文本请求响应为主，HTTP/2 引入了多路复用和头部压缩，HTTP/3 基于 QUIC 进一步降低握手和队头阻塞的影响。

八、HTTPS 比 HTTP 多了什么

HTTPS 是 HTTP 与 TLS 的组合方案，用于在不可信网络环境中安全传输应用数据。

HTTPS 主要解决以下问题：

  • 机密性：防止第三方直接读取传输内容。
  • 完整性：防止传输中的数据被修改。
  • 身份认证：通过证书确认服务器身份。

TLS 握手的核心过程包括协商加密套件、验证证书、生成会话密钥。后续应用数据通常使用对称加密传输，以兼顾安全性和性能。

九、浏览器输入网址后发生了什么

浏览器访问网页时，通常经历以下步骤：

  1. 解析 URL，识别协议、域名、端口、路径和查询参数。
  2. 通过 DNS 解析获取服务器 IP 地址。
  3. 建立连接；如果使用 HTTPS，还需要进行 TLS 握手。
  4. 发送 HTTP 请求获取 HTML 文档。
  5. 服务器返回 HTML。
  6. 浏览器解析 HTML，并发现其中引用的 CSS、JavaScript、图片、字体等外部资源。
  7. 浏览器继续请求这些资源。
  8. 浏览器根据解析结果生成可显示页面，并完成首次渲染。

浏览器通常采用边下载、边解析、边渲染的方式工作，而不是等待所有资源完全下载后再统一显示。

十、Web 渲染原理入门版

Web 渲染的目标是将 HTML、CSS 和 JavaScript 描述的页面内容转换为屏幕上的可视结果。

1. 浏览器里大致有哪些部分

现代浏览器通常包含以下职责模块：

  • 用户界面模块：负责地址栏、标签页、书签和窗口管理。
  • 网络模块：负责请求和接收资源。
  • 渲染模块：负责解析 HTML 和 CSS、构建页面结构、执行布局与绘制。
  • JavaScript 引擎：负责执行脚本。
  • 图形合成模块：负责图层合成与屏幕显示。

2. DOM 是什么

HTML 被浏览器解析后，会形成 DOM 树。

DOM 是文档对象模型，表示 HTML 文档的结构化树形表示。HTML 中的元素、属性和文本会被解析为 DOM 节点，JavaScript 可以通过 DOM API 读取、修改、插入和删除这些节点。

3. CSSOM 是什么

CSS 也会被浏览器解析成结构化的数据，叫 CSSOM。

CSSOM 是 CSS 对象模型，表示样式规则的结构化结果。DOM 描述页面结构，CSSOM 描述样式规则，二者结合后才能确定每个元素最终使用的样式。

4. 渲染树是什么

渲染树是由可见节点及其计算样式组成的树结构。浏览器会根据 DOM 和 CSSOM 生成渲染树，再基于渲染树进行后续处理。

并非所有 DOM 节点都会进入渲染树。例如 display: none 的元素不会参与布局和绘制。

5. 布局、绘制、合成分别在干什么

先记一句简单的话：

  • 布局：决定元素放哪儿、多大。

  • 绘制：把元素画出来。

  • 合成：把多个图层拼成最终画面。

  • 布局阶段根据页面结构、样式和盒模型计算每个元素的位置与尺寸。

  • 绘制阶段把元素的边框、背景、文字等转换成屏幕上的图像内容。

  • 合成阶段将多个图层按照顺序组合成最终显示结果。

布局、绘制和合成是浏览器渲染管线中的三个关键阶段。布局关注几何信息，绘制关注像素内容，合成关注图层结果的最终输出。

6. 重排和重绘

当元素的位置、大小或几何属性发生变化时，浏览器通常需要重新计算布局，这一过程称为重排或回流。

当元素的几何信息不变，但颜色、背景、阴影等外观属性变化时，通常只需要重新绘制，这一过程称为重绘。

重排通常比重绘成本更高，因为重排可能影响多个后续元素的布局结果。

十一、总结

计算机网络负责数据在不同设备之间的传输，互联网是其大规模工程化形态，Web 渲染负责将网络获取到的资源转换为浏览器中的可视页面。

整个系统可以概括为一条连续链路：

浏览器通过 DNS 找到服务器地址，通过 TCP 和 TLS 完成可靠且安全的数据传输，通过 HTTP 获取页面资源，再通过 DOM、CSSOM、布局、绘制和合成将网页呈现在屏幕上。

该链路中每一步都有明确职责，理解这些职责是学习网络与前端底层原理的基础。
