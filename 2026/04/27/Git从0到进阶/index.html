Git 核心学习笔记

Git 是目前世界上最先进的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

一、 核心概念

在操作之前，需要理解 Git 的四个工作区域：

  • Workspace (工作区)：你在电脑里能看到的目录。
  • Index / Stage (暂存区)：临时保存修改，准备提交。
  • Repository (本地仓库)：保存所有版本历史的地方。
  • Remote (远程仓库)：托管在云端的代码库（如 GitHub/GitLab）。

八、 提交建议

  1. 原子化提交：一次提交只解决一个问题或实现一个功能。
  2. 规范化日志：使用 feat:, fix:, docs: 等前缀标识提交类型。
  3. 先拉后推：在 push 之前，先执行 pull 以确保代码同步，减少冲突。
